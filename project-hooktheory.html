Case Study

Hooktheory (Client Project)

A research-driven UX audit and information architecture redesign for Hooktheory, focused on improving clarity, navigation, and learning flow for musicians, producers, and educators.

Role
UX Research • Heuristics • IA
Focus
Clarity • Findability • Cognitive load
Artifacts
Audit • Recommendations • Design guide

Overview

Hooktheory provides interactive tools for learning music theory, but the experience needed stronger structure to support comprehension and reduce friction. This project focused on identifying usability issues, clarifying navigation patterns, and creating recommendations that could better align the product with how users learn.

The Problem

Users came to Hooktheory to learn quickly, explore music concepts, and move between tools with confidence. But dense information, inconsistent navigation patterns, and overlapping interface elements increased cognitive load and made it harder to find the right next step.

The challenge was to make the learning experience feel clearer and more intuitive without losing the depth that made the product valuable to musicians and educators.

Goals

  • Improve information architecture across TheoryTab and related learning paths.
  • Reduce cognitive load for musicians, producers, and educators using the platform.
  • Create more consistent UI patterns through a lightweight design guide.

Process

My work focused on research synthesis, heuristic evaluation, and information architecture analysis to identify where the experience created friction for learners. From there, I translated those findings into clearer navigation patterns, structural recommendations, and a lightweight design guide to support consistency.

Heuristic Evaluation UX Audit Information Architecture Research Synthesis Design Guide

Key Insights

Dense pages slowed learning

Users had to process too much information before they could act, which increased effort and reduced momentum.

Navigation did not match user expectations

Category structure and movement between tools were not always aligned with how learners expected to explore music concepts.

Terminology needed to support clarity

Labels worked best when they were direct and instructional, especially in a learning environment.

Consistency supports trust

Reusable patterns and clearer visual hierarchy helped make the experience feel more predictable and easier to learn.

Recommendations

Based on the audit and synthesis work, I developed a set of recommendations focused on reducing cognitive load, clarifying navigation, and creating more consistent learning patterns across the experience.

Clarify navigation pathways

Reorganize key entry points so users can move between TheoryTab and related tools with less friction.

Reduce information overload

Improve hierarchy, spacing, and content grouping so learning can begin faster and with less visual effort.

Create a lightweight design guide

Define reusable patterns for layout, components, and labels to make the interface more consistent over time.

Outcomes & Reflection

This project strengthened my ability to work from research into structural UX recommendations. It reinforced how much learning products depend on clarity: when users are trying to understand new concepts, every navigation decision and interface label carries extra weight.

If the project continued, I would validate the revised information architecture and navigation recommendations through usability testing focused on task completion, time to find, and perceived ease of learning.
