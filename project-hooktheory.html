Case Study

Hooktheory (Client Project)

Research-driven UX audit and IA improvements for music theory learning tools—focused on clarity, navigation, and cognitive load.

Role
UX Research • Heuristics • IA
Focus
Clarity • Findability • Cognitive load
Artifacts
Audit • Recommendations • Design guide

The Problem

Users come to learn quickly, but information density and navigation complexity can slow comprehension and create friction.

Goals

  • Improve information architecture for TheoryTab / related pages.
  • Reduce cognitive load for musicians, producers, and educators.
  • Create consistent UI patterns + a lightweight design guide.

Research & Insights

Overloaded pages

Dense content increases effort before learning begins.

IA mismatch

Users’ mental models didn’t match category structure.

Terminology

Clear labels beat clever labels when learning is the goal.

Consistency wins

Reusable components reduce friction and speed trust.

Design Exploration

Outcomes

Summarize key recommendations delivered + what success metrics you’d track (task completion, time to find, error rate).
